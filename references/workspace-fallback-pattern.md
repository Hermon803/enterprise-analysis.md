# 工作区回退模式（Workspace Fallback）

当企业分析的 web 搜索被拒绝或网络不可达时，使用本回退流程。

## 触发条件

- 用户拒绝网络请求（blocked commands）
- 网络不可达（timeout / DNS failure）
- delegate_task 搜索工具不可用

## 回退流程

1. **搜索用户工作目录** — 使用 `search_files` 查找既有的分析文件：
   ```
   search_files(target='files', pattern='*公司名*')
   search_files(target='files', pattern='*公司简称*')
   search_files(target='content', pattern='*公司全称*')
   ```

2. **优先读取 Markdown 文件**（.md），其次是 DOCX（需转换），PDF 最后考虑。

3. **提取结构化数据** — 按照六维维度从既有文件中抽取：
   - 企业介绍与主营业务 → 定位行、企业标签
   - 核心人物 → 创始人/CEO 段落
   - 技术与知识产权 → 专利数、软著、研发平台、标准参与
   - 经济贡献与经营数据 → 表格数据
   - 供应链与区域布局 → 依赖/风险/布局信息
   - 竞争与市场结构 → 份额、排名、竞争格局

4. **标注数据来源** — 在报告中明确标注：
   ```
   > 数据来源：基于[路径]既有分析文件提取，原数据来源于[原信源列表]
   > ⚠️ 外部网络检索不可达，以下数据未经增量交叉验证
   ```

5. **记录缺失维度** — 对既有文件中未覆盖的维度，标注"本维度数据待补充"。

## 实战案例：深圳格芯集成电路装备

- 路径：`~/Progress/analysis/v1.1/深圳格芯集成电路装备/格芯集成电路_产业安全分析.md`
- 提取到的 IP 数据：139项发明及实用新型专利、40余项发明专利、14项软著、参与1项国标、承担02专项两个课题
- 发现的数据歧义："139项发明及实用新型"与"40余项发明专利"的关系不明确（包含关系 vs 独立统计）
  处理方式：在报告中如实标注歧义，不做归并假设
- 缺失数据：PCT申请数、海外专利布局、近三年专利增长趋势

## 适用场景

- 用户在同一个工作区进行多轮企业分析，之前的结果可直接复用
- 用户已有第三方研究机构提供的行业分析文档
- 网络不稳定的离线工作环境
