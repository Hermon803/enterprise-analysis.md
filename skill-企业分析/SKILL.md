# 企业分析 · AI产业链企业分析

## 约束规则（必须遵守）

1. **维度锁定**：最终报告有且仅有以下六个维度，不得增减、合并或扩展：
   ① 企业介绍与主营业务 / ② 核心人物 / ③ 技术与知识产权 / ④ 经济贡献与经营数据 / ⑤ 供应链与区域布局 / ⑥ 竞争与市场结构
2. **制品固定**：每个分析必须生成且仅生成三种文件：
   - `[公司名]_企业分析.md` — Markdown 报告
   - `[公司名]_企业分析.pdf` — PDF 报告（含图）
   - `[公司名]_企业分析.docx` — Word 报告
3. **数据溯源**：每项数据必须标注来源，禁止编造，多源交叉验证。
4. **篇幅限制**：PDF最终产物不超过两页A4。所有内容（含图片）必须压缩在2页内，表格≤6行，每个维度正文≤3行，精炼优先。
5. **相关产品展示**：报告底部（数据源上方）增设"相关产品"视觉展示区，展示1-2个最具代表性的产品图片。这是视觉/版面元素，不属于分析维度，不违反第1条的维度锁定。纯互联网公司无实体产品时，展示代表性App图标（使用simpleicons或官网下载）。
6. **模板性质**：下方"报告模板"为内容参考模板，PDF实际排版需按两页限制做密度优化，并非逐字逐段输出。

## 用法

```bash
/企业分析 <公司名> <官网域名> <英文名>
```

**示例：**
```bash
/企业分析 百度 baidu.com baidu
/企业分析 NVIDIA nvidia.com nvidia
/企业分析 比亚迪 byd.com
/企业分析 深圳格芯集成电路装备 grandtec-ic.com
```

```
├── [公司名]_企业分析.md      ← 必出
├── [公司名]_企业分析.pdf      ← 必出

```

## 阶段Ⅰ：企业六维分析

### ① 企业介绍与主营业务

**收集：** 企业全称、成立时间、总部所在地、企业性质（国企/民企/外企/合资）、上市情况、企业标签（专精特新/小巨人/独角兽/瞪羚/百亿千亿/高新技术企业等）、主营业务、产业链定位

**输出格式：**
```markdown
> **定位**：[企业类型] | [上市/未上市] | [城市] [成立年份] | [企业标签]
> **主营业务**：[一句话概括核心产品/服务]
```

### ② 核心人物

**收集：**
- 创始人/CEO姓名
- 籍贯/出生地
- 毕业院校及专业
- 关键履历（重点：白手起家/技术背景/行业贡献等，1-2句话）

**输出格式：**
```markdown
> **创始人/CEO**：[姓名] · [籍贯] · [毕业院校] · [关键履历一句话]

[图片:founder: 人物描述]
```

### ③ 技术与知识产权

**收集：** 国产化率、核心依赖（海外开源/算力/芯片）、有效专利数及发明占比、PCT布局、参与国标/行标情况

**输出格式：**
```markdown
> **国产化率**：[X%] · **核心依赖**：[海外/自研/混合]
> **知识产权**：累计专利[X]项 · 发明占比[X]% · PCT[X]项
> **标准参与**：国标[X]项 · 行标[X]项 · [牵头/参与]
```

### ④ 经济贡献与经营数据

**收集：** 最新年度营收/产值、增速、税收贡献、就业人数、带动上下游企业数、利润/毛利率

**输出格式：**
```markdown
| 指标 | 数据 | 同比/说明 |
|------|------|---------|
| 营收/产值 | | |
| 税收贡献 | | |
| 直接就业 | | |
| 带动上下游企业 | | |
```

### ⑤ 供应链与区域布局

**收集：** 上游芯片供应商及出口管制影响、国产替代进展、数据库/中间件/OS替代方案、总部/研发/生产所在地、外迁扩建计划

**输出格式：**
```markdown
> **芯片依赖**：[供应商/型号] · **替代方案**：[进展]
> **供应链评级**：[安全/低风险/中风险/高风险]
> **布局**：总部[X] · 研发[X] · 生产[X]
> **外迁/扩建**：[有/无，说明]
```

### ⑥ 竞争与市场结构

**收集：** 国内/海外市场份额排名、竞争格局类型、对中小企业生存空间影响

**输出格式：**
```markdown
> **国内**：[份额]（第[X]）· **海外**：营收占比[X]%
> **竞争格局**：[类型] | 对中小企业影响：[挤出/带动/中性]
```

## 阶段Ⅱ：图片搜索与组装

### Step 1 — 标注图片需求

在阶段Ⅰ报告中，用占位标记标注需要图片的位置：

```markdown
[图片:logo: 公司Logo描述]
[图片:founder: 创始人/CEO人物描述]
[图片:product: 核心产品描述]
```

**示例：**
```markdown
[图片:logo: NVIDIA绿色方形Logo，含"NVIDIA"字样]
[图片:founder: 黄仁勋(Jensen Huang)半身照，身着黑色皮衣]
[图片:product: NVIDIA Blackwell B200 GPU芯片渲染图]
```

### Step 2 — 图片抓取（强制执行，无图不交付）

**图片是报告的必需组成部分，严禁跳过。** 三张图缺一不可。

首选用配套脚本全自动抓取：

```bash
python3 scripts/fetch_images.py \
  --company "$公司名" \
  --domain "$域名" \
  --english "$英文名" \
  --founder "$创始人名" \
  --output ./image/
```

脚本自动执行以下流程：
- **Logo**: simpleicons.org → 官网HTML解析 → 常见路径
- **创始人**: 百度百科 → 官网关于我们页面
- **产品图**: 官网产品页 → 产品详情子页面

脚本特性：自动去重、图片有效性校验（PIL verify）、跳过HTML响应、User-Agent轮换。

全部失败才标注"图片待补充"。

**Logo 抓取（手动备选）：**

| 优先级 | 方法 | 说明 |
|--------|------|------|
| ① | `curl cdn.simpleicons.org/<英文名>` | SimpleIcons CDN（Cloudflare），国内可达 |
| ② | curl 官网 `grep -i icon` | 从官网HTML提取favicon或og:image |
| ③ | 手动搜索百度百科 + curl | 搜"公司名 logo 百度百科" |

**创始人/CEO照片抓取（手动备选）：**

| 优先级 | 方法 | 搜索词示例 |
|--------|------|-----------|
| ① | 百度百科（最高优先级，使用curl绕过403） | `林宜龙` 或 `任正非` |
| ② | 官网关于我们/团队页面 | 过滤logo/icon类图片 |

**产品图抓取（手动备选）：**

| 优先级 | 方法 | 说明 |
|--------|------|------|
| ① | 官网产品页提取 | 支持 data-src 懒加载 |
| ② | 产品详情子页面提取 | 点进具体产品页找大图 |

关键技巧：
- **中文搜索优先**：国内科技媒体图片走国内CDN加速，下载成功率高
- 对懒加载网站，检查 `data-src`、`data-original` 属性
- 图片URL优先选**大分辨率**（含 `large`/`high`/`1920` 等关键词）

**⚠️ 关键发现：WebFetch ≠ 真实网络。** WebFetch 工具有域名安全策略，会拒绝访问 nvidia.com 等知名站点。但 **curl/Bash 不受此限制**。图片抓取阶段应**全部用 Bash(curl) 下载**。

**国内可达源 ✅（优先使用）：** `cdn.simpleicons.org` · 百度系 · 国内科技媒体 · 公司官网（通过curl）
**国内不可达源 ❌：** `upload.wikimedia.org` · Google系 · `logo.clearbit.com` · `raw.githubusercontent.com`

```bash
# 图片有效性检查
curl -sL <图片URL> -o /tmp/img_check
file /tmp/img_check   # 应输出 "PNG/JPEG/SVG image data"
# 如输出 "HTML document" → 非图片，跳过
```

### Step 3 — 图片嵌入与组装

```bash
# base64编码
LOGO_B64=$(base64 -w0 image/logo.*)
FOUNDER_B64=$(base64 -w0 image/founder.*)
PROD_B64=$(base64 -w0 image/product.*)

嵌入报告时使用 `[图片:类型:描述]` 占位标记：
- `[图片:logo:xxx]` → h1同一行右上角，height=28px，右浮动（不单独占行），两页版必用此模式
- `[图片:founder:xxx]` → "核心人物"板块右侧，width≤100px（两页版≤80px），圆形裁切
- `[图片:product:xxx]` → 用于"相关产品"展示区（PDF底部），width≤140px，2图并排居中；两页版product图移至底部"相关产品"区域
- `[图片:app:xxx]` → 互联网公司App图标，width≤60px，与产品图同样式统一展示
```

### Step 4 — 生成最终文件

| 格式 | 命令 |
|------|------|
| PDF | 自包含HTML → `weasyprint report.html report.pdf` |
| Word | Markdown → `pandoc report.md -o report.docx` |

**PDF排版要求：**
- A4纸张 · 边距 **1.5cm**（不低于1.3）· 中文使用 Noto Sans CJK SC
- h1公司名与logo flex同行（h1左 logo右），h1字号**22pt**品牌色，带2px字间距
- h2板块标题字号**12pt**，加1px字间距，上间距0.7em
- blockquote（正文容器）字号9.5pt，行高**1.85**，内边距**0.5em 0.9em**，文字不贴边
- 表格字号9pt，区分"数据"与"说明"两列（避免纯百分比），行高保持1.7以上
- 关键数据用红色加粗高亮
- 去掉所有`---`分隔线，板块间用**0.7em**上下间距自然分隔
- **相关产品展示区**（底部，数据源上方）：标题"相关产品" 12pt，产品图2个并排居中，width≤140px，间距1em。纯互联网公司用App图标（width≤60px），同样式排列。
- 数据源区块置底，字号7.5pt，浅灰色，上边线分隔

### 图片兜底规则

- 所有图片源遍历后仍无结果 → 跳过图片，纯文字输出
- PDF中标注 `[图片待补充]` 占位区域
- 不编造、不使用侵权图片

## 环境依赖

| 工具 | 用途 | 安装 |
|------|------|------|

```powershell
msedge --headless --print-to-pdf=report.pdf report.html
```
